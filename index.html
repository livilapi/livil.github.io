    Request signature, all parameter pairs are sorted in ascending order by dictionary (abcd ...), and the secret assigned by the platform is the key. First, all parameters except the signature itself need to be added in the format "KEY = value" and use the character "&". Add the exact order of the request, then use md5 to encrypt the generated string and then lowercase it. Note: The sign parameter does not participate in the signature; the null parameter does not participate in the signature.

function post($url, $fields){
  $herder = array("Content-Type: application/json","Accept-Language: en-US");
  $ch=curl_init();
  curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);
  curl_setopt($ch, CURLOPT_URL, $url);
  curl_setopt($ch, CURLOPT_HTTPHEADER, $herder); 
  curl_setopt($ch, CURLOPT_CUSTOMREQUEST, "POST"); 
  curl_setopt($ch, CURLOPT_POST, true); 
  curl_setopt($ch, CURLOPT_POSTFIELDS, $fields); 
  curl_setopt($ch, CURLOPT_TIMEOUT, 3); 
  $response = curl_exec($ch);
  curl_close ($ch);
  return $response;
}

    HTTP Request

    POST /pay/order/create

    Sample signature

    appKey: kpecb34edd45cf4d2bb42229aac78f66c5

    appSecret: sbf54f0ae60704c69859b028c5cdb1fbd

    Request header

    Parameter Default Description
    Content-Type String application/json
    Accept-Language String Client language type: zh-CN,en-US,ko-KR

    Query Parameters

    Parameter Default Description
    createType Integer(*) Create order type (1: Fiat currency payment (fiat currency type and payment amount are required) 2: Limited payment currency (currency ID, payment quantity is required) 3: Limited receipt currency (receiving currency set in the merchant background, payment amount Required))
    currencyType Integer Type of legal currency (1: CNY, 2: USD 3: KRW)
    amount Float Fiat currency payment amount
    coinId Integer Currency ID(Get the currency list of the payment currency.)
    coinNum Float Number of currencies
    merchantOrderNum String(*) Merchant order number（unique）
    goodsName String product name
    ts Long(*) Timestamp (seconds)
    appKey String(*) Interface unique private appKey.
    sign String(*) Request a signature, and all parameter pairs are sorted in ascending dictionary order (for example: a=c&b=g&F=j...), add &APPSECRET=key at the end of the sorting. When sorting, you need to convert the parameter name to uppercase, and then use md5 for the string Encrypt, then convert it to lowercase

    Response format

    Parameter Default Description
    code Integer Return code 200: success 201: Failure
    msg String Description
    data Object/Array Return data, object or array { "Code": 0, "Msg": "success", data: {} }

    Get the currency list of the specified payment currency.

    HTTP Request

    POST /pay/order/coin/list

    Request header

    Parameter Default Description
    Content-Type String application/json
    Accept-Language String zh-CN, en-US,ko-KR

    Query Parameters

    Parameter Default Description
    appKey String Interface unique private appKey.
    ts Long Timestamp (seconds)
    sign String Request a signature, and all parameter pairs are sorted in ascending dictionary order (for example: a=c&b=g&F=j...), add &APPSECRET=key at the end of the sorting. When sorting, you need to convert the parameter name to uppercase, and then use md5 for the string Encrypt, then convert it to lowercase

    Response format

    Parameter Default Description
    code Integer Return code 200: success 201: Failure
    msg String Description
    data Object/Array Return data, object or array { "coinId": 1, "coinName": "BTC" }

    Callback description

    Parameter Default Description
    appKey String Interface unique private appKey.
    ts Long Timestamp (seconds)
    sign String Parameter signature (passed parameters, the caller needs to be combined with AppSecret for signature verification)
    qrCodeType Integer QR code type
    merchantOrderNum String Merchant order number
    payOrderNum String Payment order number
    goodsName String product name
    status Integer 1. The default state 2. To be paid 3. The payment is successful, 4. Payment failed 5. Payment cancelled
    receiveCoinNum Float Receiving currency quantity
    receiveCoinName String Receiving currency name

    Errors

    The API uses the following error codes:

    Code Meaning
    200 successful.
    201 The operation failed.
    202 data is abnormal.
    203 Signature verification failed.
    206 The number of visits exceeds the limit, and the visit is denied.
    207 No permission to access.
    208 The request expired.
    400 The request parameter is wrong.
    405 The account is disabled.
    407 The client type verification failed
    408 The payment password is incorrect.
    415 Unsupported Media Type.
    500 Service call exception.
